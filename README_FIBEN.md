# SchemaLens Artifact — FIBEN Case Study

This README describes how to reproduce the FIBEN part of the SchemaLens evaluation.

FIBEN is used in the paper as a finance-oriented comparative dataset. It complements the IMDb running example by testing SchemaLens on business-oriented structures, including companies, officers, financial metrics, securities, holdings, transactions, and customer accounts.

The FIBEN case study supports reproduction of:

1. scale-factor preparation or reuse;
2. methodology execution over the FIBEN conceptual/workload schema;
3. MongoDB candidate generation and benchmark execution;
4. result analysis across scale factors;
5. FIBEN representative rows in the cross-dataset summary.

Data availability

Large FIBEN scale-factor datasets are not stored directly in this Git repository because of size constraints.

The prepared FIBEN scale-factor package used in the paper is available on OSF:

https://osf.io/532rn/overview?view_only=0a93fbed1db745d0978aa2e9f6cd7c78

This package should be used by reviewers who want to reproduce the benchmark or verify the analysis without regenerating the scale factors from scratch.

After downloading and extracting the OSF package, organize the data locally as:

data/
  fiben/
    sf1/
    sf10/
    sf30/

The three scale factors are:

sf1   -> workload-induced FIBEN baseline subset
sf10  -> scaled dataset generated from sf1
sf30  -> scaled dataset generated from sf1
Two reproduction options
Option A — Use the prepared OSF scale-factor package

This is the recommended path for reviewers.

Download the prepared FIBEN scale-factor package from OSF.
Extract it locally using the folder structure shown above.
Run the methodology notebook.
Run the MongoDB benchmark script using the extracted scale-factor folders.
Run the analysis notebooks to regenerate the paper-support results.

This option avoids the time-consuming scale-factor generation step.

Option B — Regenerate the scale factors

Use this option if you want to reproduce the FIBEN scale-factor generation process.

Run:

scale_generator/fiben/Fiben_generate_scale_factor.ipynb

This notebook documents the generation of:

sf1
sf10
sf30

Additional scale-generation instructions are provided in:

scale_generator/fiben/README.md

Benchmark reproduction

The benchmark runner is:

benchmark/fiben/run_fiben_mongo_benchmark.py

The benchmark should be executed with:

the FIBEN data directory;
the benchmark artifacts directory generated by the methodology;
the results directory;
the scale label.

Example command for sf1:

python benchmark/fiben/run_fiben_mongo_benchmark.py \
  --data-dir data/fiben/sf1 \
  --artifacts-dir analysis/fiben \
  --results-dir results/fiben/sf1 \
  --scale-label sf1 \
  --mongo-host 127.0.0.1 \
  --mongo-port 27018 \
  --mongo-username mongo \
  --mongo-password mongo \
  --mongo-auth-source admin \
  --batch-size 100000 \
  --repetitions 10 \
  --force-rebuild-db \
  --verbose

Repeat the execution for:

sf10
sf30

MongoDB can be started from the repository root with:

docker compose up -d

The default MongoDB port is:

27018

Paper connection

The FIBEN artifact supports the cross-dataset evaluation in the paper.

The representative FIBEN cases used in the cross-dataset summary include:

Q2_CompanyWithIndustryCountryAndListedSecurities
Q3_SecuritiesHeldInEachFinancialServiceAccount
Q5_ReportsAndMetricDataOfCompany

These queries test whether SchemaLens can activate meaningful MongoDB configuration families in a finance-oriented workload with association, associative, and analytical/containment-like access patterns.

Lightweight verification

To verify the FIBEN part of the paper without rerunning the full benchmark:

use the prepared OSF scale-factor package if data are needed;
use the analysis notebooks in analysis/fiben/;
use the benchmark aggregate outputs generated by the benchmark runner.

Full benchmark reproduction is supported, but it is more time-consuming because it requires materializing MongoDB candidate configurations and executing repeated cold/hot benchmark runs for each scale factor.
