Esse não é bem o readme mas são anotações importantes para que alguém possa reproduzir o que foi feito.

A execução começa com a definição do dataset base ( em cima dele é feita todo o framework), os outros scale facote só testam as mesmas configurações geradas com volumes maiores.

Então para cada dataset é necessário rodar o framework que se encontra na pasta methodology separadamente. Esse script irá gerar vários artefatos que são necessários para a execução do benchmark e principalmente as configurações do mongodb que serão testadas.

A pasta scale_generation contém os scripts que foram usados para gerar os scale fator dos datasets do imdb e fiben, pois o dataset do ldbc_snb é obtido diretamente dos arquivos do site do benchmark (se você souber coloque quais são os principais arquivos que usamos, senão me pergunte e eu falo ).

Os arquivos na pasta benchmark devem ser executados no servidor e dependem dos artefatos gerados pela metodologia, principalmente do arquivo que gera as configurações do mongo que irão ser testadas.
