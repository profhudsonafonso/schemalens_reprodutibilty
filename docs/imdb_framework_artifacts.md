# IMDb Framework Artifacts

This document describes the artifacts generated by the IMDb framework step and consumed by the IMDb MongoDB benchmark runner.

## Purpose

The IMDb framework step transforms the workload and framework rules into benchmark-ready MongoDB experiment artifacts.

The benchmark runner consumes two main files:

    benchmark/imdb/mongo_experiment_catalog.csv
    benchmark/imdb/benchmark_execution_template.csv

## Main benchmark input artifacts

### mongo_experiment_catalog.csv

Path:

    benchmark/imdb/mongo_experiment_catalog.csv

Purpose:

    Defines the MongoDB experiment configurations to be materialized and benchmarked.

Expected current size:

    28 lines, including header
    27 experiment rows

Expected header:

    experiment_id,config_name,activated_class,benchmark_family,scale_factor,scale_label,execution_phase,mongo_db_name,selected_root,primary_collection,embedding_depth,embedding_variant,embedded_entities,snapshot_entities,referenced_entities,changed_region_entities,derived_from_queries,n_supporting_queries,primary_queries,secondary_affected_queries,control_queries,n_primary_queries,n_secondary_affected_queries,n_control_queries,experiment_goal

### benchmark_execution_template.csv

Path:

    benchmark/imdb/benchmark_execution_template.csv

Purpose:

    Defines the planned benchmark execution rows.

Expected current size:

    5401 lines, including header
    5400 planned execution rows

Expected header:

    benchmark_run_id,experiment_id,config_name,activated_class,benchmark_family,scale_factor,scale_label,mongo_db_name,selected_root,primary_collection,embedding_depth,embedding_variant,embedded_entities,snapshot_entities,referenced_entities,query_name,query_group,run_phase,phase_order,repetition,execution_status,error_message,start_ts,end_ts,latency_ms,success,documents_returned,documents_written,bytes_read_estimate,bytes_written_estimate,experiment_goal,query_group_order

## Trace artifacts

Additional framework outputs should be written to:

    analysis/generated/framework/imdb/

These files are useful for inspection and traceability, but are not directly consumed by the IMDb benchmark runner.

Examples include:

    analytical_matrix_final_df.csv
    document_variable_matrix_df.csv
    mongo_configuration_catalog_df.csv
    mongo_experiment_catalog.csv
    benchmark_execution_template.csv
    benchmark_execution_plan_summary_df.csv
    framework_artifact_manifest.json

## Makefile target

The intended Makefile target is:

    make imdb-framework IMDB_SF_ROOT=<path-to-imdb-sf-outputs> IMDB_ACTIVE_SCALE=sf0.25

The target should generate or refresh:

    benchmark/imdb/mongo_experiment_catalog.csv
    benchmark/imdb/benchmark_execution_template.csv

and save trace files under:

    analysis/generated/framework/imdb/


## Framework execution script

The IMDb framework execution wrapper is located at:

    methodology/run_imdb_framework_notebook.py

It executes:

    methodology/imdb_methodology.ipynb

and exports the benchmark-ready artifacts to:

    benchmark/imdb/mongo_experiment_catalog.csv
    benchmark/imdb/benchmark_execution_template.csv

Trace artifacts are written to:

    analysis/generated/framework/imdb/

The Makefile entry point is:

    make imdb-framework IMDB_SF_ROOT=<path-to-imdb-sf-outputs> IMDB_ACTIVE_SCALE=sf0.25
